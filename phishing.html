Proposed Solution for Catching Phishing

Written: November 2020

Phishing is a type of spam where the message sender pretends to be a trustworthy or legitimate user in order to trick other users into handing over their personal information by asking them to complete some action such as password verification, or updating other PII.

We suggest using Google’s Safe Browsing API which keeps a list of phishing URLs. This solution does not required the online education platform to share any personally identifiable information with Google. Our illustrated above, is as follows:

  1. First the online education platform extracts URLs from the user generated content.
  2. Then the platform providers maintain a local database containing URLs to phishing sites. The URL list is obtained from Google’s Safe Browsing API. The database will be periodically updated, so that new changes can be incorporated and the database remains up to date. An advantage of keeping a local database is that data is exchanged infrequently with the Google server so the server never knows the actual URLs which are being queried. Whereas if you use the live API, then Google’s server will know which URLs you are checking.
  3. A tool will have to be specifically designed for the intended platform, which will check each URL extracted from the user posts against the local database of phishing URLs. If the URL is a phishing one, it will be removed and appropriate action can be taken against the user who posted it.
